## 기술 스펙

### 프론트엔드: HTML / CSS / JavaScript

웹의 가장 기본적인 구성 요소이기에 이번 과제인 로그인/ 회원가입 기능에 알맞다고 생각했고, CNU 학회에서 프론트엔드 기초로 배우고 있기에 이해하기에 낫다고 생각해서 선택했다.

### 백엔드 : Node.js / Express

이전에 Supabase를 사용한 적은 있었지만, 직접 백엔드 구현은 처음이었기에 입문자가 시작하고 쉬운 Node.js + Express 조합을 이용했다.

DB : SQQLite

처음 구현하는 프로젝트이고 규모가 크지 않았기에 가볍고 빠르게 시작가능한 SQLite가 적합하다고 생각했다.

### 프록시 / 웹서버 : Nginx

Nginx는 이번 과제 하면서 처음 접했는데, 이해한 바로는 사용자의 요청을 내부 Express 서버로 보내기 전 한번 거치는 앞단의 서버라고 이해했다.

### 배포 / 도메인

실제로 서버를 빌려 도메인 연결 후 배포는 처음이기에 배울 점이 많았다. 무료로 시작가능한 AWS를 통해 아마존 리눅스 환경에서 배포를 했다. 도메인 연결 후 HTTPS 까지 적용하면서 실제 웹서비스처럼 접근 가능하게 만들어 보았다.

### 전체 구조

**브라우저 -> Nginx -> Express 서버 -> DB(SQLite) -> 브라우저**

조금 더 자세히 설명하면 다음과 같다. 브라우저에서 회원가입, 로그인등 요청을 보냄. Nginx가 요청을 먼저 받아 Express 내부 서버로 전달함. Express 서버에서 실제 기능을 처리함. 필요한 경우 SQLite 데이터베이스에서 사용자 정보를 저장, 조회함 다시 처리 결과를 브라우저에 응답함.

### 배포 과정

1. 서버 환경을 준비했다.
2. Node.js / Express 서버가 실행되도록 설정했다.
3. SQLite를 이용해 회원 정보를 저장하도록 구성했다.
4. Nginx를 통해 외부 요청을 Express 서버로 전달하도록 설정했다.
5. 도메인을 연결 후 HTTPS를 적용함. 로컬 환경에서만 웹 편집을 하다, 실제 서버에 올리니 처음 안 개념들도 많았다. 그래도 배포를 직접 해보면서 웹서비스가 어떻게 공개되어 작동하는지 조금 더 이해할 수 있게 되었다.

### 막혔던 부분 및 해결 과정

1. 백엔드 & 서버 구조 개념이 잘 잡히지 않았던 점 이전에 BaaS를 사용했기에, 직접 서버를 두고 회원가입 및 로그인 처리하는 구조를 잘 몰랐다. LLM과 모르는 개념들을 익히면서 브라우저 → Nginx → Express 서버 → DB → 브라우저 라는 기본적인 전체 흐름을 이해하게 되었다.
2. 리눅스 환경이 처음이라 어려웠던 점 직접 서버를 구축하면서 리눅스 환경도 처음 사용해보았다. 윈도우환경만 사용하다 처음 리눅스를 접하니 낯설었다. 특히 회원가입 시 DB에 가입 정보가 제대로 저장되지 않는 것처럼 보이는 문제가 있었다. 리눅스 서버에 필요한 모듈 대신 윈도우용 모듈이 설치되어 있어 오류가 발생한 것을 알고 고치게 되었다.
3. 구현 방식에 대한 고민 처음에는 HTML 파일과 JS를 LLM을 통해 개념을 이해하고 구축하려했는데 생각보다 시간 소요가 커서, 우선 바이브 코딩을 통해 웹을 구현하고, LLM을 통해 이해하는 방법을 채택했다.
